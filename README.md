# Sky Blaster — 기말 발표

---

## 1. 게임에 대한 간단한 소개

> **종스크롤 슈팅 × 로그라이크** 
**Sky Blaster** 는 종스크롤 슈팅 장르와 로그라이크 시스템을 결합한 안드로이드 모바일 게임이다. 사용자는 화면을 터치한 채 드래그해 캐릭터를 자유롭게 이동시키고, 총알은 자동으로 발사된다. 일반 스테이지에서 몰려오는 적을 처치하여 경험치를 얻어 레벨업을 해 랜덤한 보상을 획득할 수 있고, 정해진 시간마다 보스 스테이지에 진입할지 계속 레벨업을 할 지 선택할 수 있다. 보스 스테이지에서는 보스와 싸우며 보스를 잡으면 최종 클리어하는 게임이다.

게임의 전반적인 시스템은 다음과 같다.

- **플레이어 조작** : 자동 발사, 드래그 이동, 터치 스킬
- **레벨업 보상** : 능력치 증가, 특수 무기, 스킬
- **몬스터** : 자폭 / 원거리 / 분열 / 보스 몬스터
- **종료 지점** : 플레이어 사망 / 보스 처치
- **재미 포인트** : 로그라이크를 통해 내가 원하는 빌드를 개척하고, 보스를 잡을 타이밍을 직접 선택함으로써 불합리한 상태에서 싸우는 불상사를 막는다.

## 3. 사용된 기술 · 참고한 것 · 수업 내용에서 차용한 것 · 직접 개발한 것

### 3-1. 사용된 기술


**렌더링 · 게임 루프**

- **`Canvas` 2D 직접 렌더링** : `android.graphics` 의 `Canvas` / `Bitmap` / `Paint` / `Rect` / `RectF` 만으로 모든 화면을 그린다. 기체·적·탄·이펙트는 `drawBitmap`, 경험치 구슬은 `drawCircle`, HUD 는 `drawText` / `drawArc` / `drawRect` 를 직접 호출.
- **frameTime 기반 루프** : 매 프레임 모든 오브젝트의 `update(gctx)` 후 `draw(canvas)` 호출. 이동·쿨다운·수명을 전부 `값 += 속도 * gctx.frameTime` 형태로 처리해 프레임 레이트가 달라도 같은 속도 유지.
- **레이어 기반 그리기 순서** : `MainScene.Layer` 13종 enum (BACKGROUND → PLAYER → BULLET → ENEMY → ENEMY_BULLET → LASER → CONTROLLER → STARS → EXP_ORB → VFX → UI → SKILL_BUTTON → PAUSE_BUTTON) 을 `World` 가 순서대로 그려 우선순위를 보장.

**해상도 독립 좌표계**

- **가상 좌표계 + Matrix 변환** : `GameMetrics` 가 고정 가상 해상도를 실제 화면 크기에 맞춰 `transformMatrix` 로 스케일·정렬하고, `canvas.withMatrix()` 안에서 모든 그리기를 수행해 기기 해상도와 무관하게 같은 화면 보장.
- **터치 좌표 역변환** : 입력은 `metrics.fromScreen()` 으로 실제 좌표를 가상 좌표로 되돌려 버튼·드래그 판정에 사용.

**충돌 처리**

- **AABB 박스 충돌** : `IBoxCollidable` + `collidesWith` 로 두 사각형 겹침 검사.
- **히트박스 축소** : 보는 스프라이트보다 판정을 작게 잡아 체감을 좋게 함 (예: 보스 본체 `COLLISION_INSET_RATIO = 0.75`, 보스탄은 타입별 비율).
- **무적/명중 중 판정 제거** : 보스는 등장·페이즈 전환 동안 `collisionRect` 를 빈 사각형으로 만들어 피격되지 않게 하고, 보스탄은 명중 이펙트 재생 중 판정을 비워 중복 피격 방지.

**메모리 관리 — 오브젝트 풀링**

- **`IRecyclable` + `World.obtain()`** : Enemy / Bullet / BossBullet / ExpOrb / DamagePopup 를 풀에서 꺼내 재사용, 죽으면 회수.
- **생성자 `private` + 정적 팩토리** : `Enemy.get()` / `Bullet.get()` / `BossBullet.get()` 만 열어둬서 항상 풀을 거치게 했다. 덕분에 후반에 적·탄이 수백 개로 늘어도 GC 부담이 거의 없다.
- **공유 비트맵 캐시** : 보스탄 명중 이펙트 시트(8종)를 `sharedHitBitmaps` 한 맵에 한 번만 로드해 공유.

**사운드**

- a2dg `Sound` (MediaPlayer 기반) 으로 BGM 루프 + 효과음 재생. `onEnter/onExit/onPause/onResume` 에 BGM 을 연동해 일시정지·화면 전환에 자연스럽게 멈추고 재개.
- **효과음 볼륨 분리** : `Sound.playEffect(resId, volume)` 에 볼륨 파라미터를 직접 추가(원본은 고정 볼륨) → 반복 발사음은 작게(0.15), 피격·힐·폭발은 크게(0.6~1.0).
- **광폭화 루프 사운드 + 에러 복구** : 별도 `MediaPlayer` looping + `setOnErrorListener` 로 끊김 자동 복구.

**애니메이션 · 연출 · 햅틱**

- **스프라이트 시트 frame animation** : 스킬 VFX·보스탄 명중 이펙트를 가로 strip 시트로 한 프레임씩 잘라 그린다 (1회성은 마지막 프레임 클램프, 지속형은 순환).
- **회전 렌더링** : 탄을 진행 방향(`atan2`)에 맞춰 `canvas.rotate()`, 보스 낫탄은 시간 비례 회전.
- **Bezier 곡선 보간** : 보스 낫탄 궤적을 2차 Bezier로 계산.
- **히트스톱 + 화면 흔들림** : 강조 순간 0.06초 `update` 정지 + 0.18초 화면 진동/줌(감쇠).
- **진동(Vibrator)** : 플레이어 피격 시 햅틱 피드백.

**앱 구조 · 입력 처리**

- **ViewBinding** : 타이틀 화면을 `ActivityMainBinding` 으로 바인딩 (`viewBinding = true`).
- **Activity 전환 · 생명주기** : 타이틀(`MainActivity`) → 게임(`SkyBlasterActivity`) 을 `Intent` 로 전환, `onResume` / `onPause` 에서 타이틀 BGM `MediaPlayer` 생성·해제.
- **터치 입력 처리** : `onTouchEvent` 의 `MotionEvent` 를 좌표 역변환한 뒤 스킬·일시정지 버튼 영역에서 먼저 가로채고, 남으면 플레이어 드래그로 위임.

**◾ 전투 시스템**

#### 플레이어 — 이동·전투 (`Player`)

- **드래그 추종 이동 (DragonFlight 차용 → 2D 확장)** — DragonFlight 는 터치한 곳의 x좌표만 목표로 잡아 좌우로만 움직였는데, 이를 가져와 손가락 방향을 단위 벡터(길이 1짜리 방향)로 만들어(`x += dx / len * SPEED * frameTime`, y 도 동일) 상하좌우 2D 전체를 따라가도록 확장했다. 프레임시간을 곱해 이동하므로 기기 성능과 무관하게 같은 속도로 움직이고, 목표에 거의 닿으면 스냅해 떨림을 막는다.
- **능력치 통합 계산** — 카드·버프·치명타로 얻은 배수를 더하지 않고 곱으로 쌓아 데미지·공속에 한 번에 반영하고, 버프는 시간이 지나면 배수를 원래대로 되돌린다. 경험치가 차면 레벨업해 보상 카드를 고른다.

#### 무기 시스템 (`sealed class Weapon`)

`Weapon` 을 4종으로 나눠 각자 발사 방식을 구현했고, 같은 무기라도 카드로 얻은 등급(고급/희귀/영웅)에 따라 발사 양상이 달라진다.

- **직진** — 단발 직진탄. 가장 기본.
- **샷건** — 펠릿 여러 발을 부채꼴로 흩뿌린다. 정면을 한가운데 두고 좌우로 같은 간격씩 각도를 벌린 뒤, 그 각도의 sin·cos 로 각 펠릿의 속도 방향을 정한다(`vx = sin(rad)*SPEED, vy = -cos(rad)*SPEED`). 등급이 높으면 펠릿 수와 퍼짐 각이 늘어난다.
- **유도 미사일** — 발사 뒤 가장 가까운 적을 추적한다. 적을 향한 '이상적인 속도'와 '현재 속도'의 차이를 매 프레임 조금씩만 메워(lerp) 급히 꺾이지 않고 부드럽게 휜다. 등급이 높으면 발수가 늘어난다.
- **레이저** — 직선 영역의 빔을 잠깐 펼쳐 그 안의 모든 적을 일정 간격으로 여러 번 때린다. 빔 안에 오래 머문 적일수록 많이 맞는다. 등급이 높으면 빔이 넓어진다.

#### 발사체 — 한 클래스로 직진/산탄/유도 (`Bullet`)

- **단일 클래스 다형 이동** — 속도·선회율·목표 속도 같은 인자만 바꿔 직진·산탄·유도를 한 클래스로 처리한다. 선회율이 있을 때만 가장 가까운 적 쪽으로 현재 속도를 매 프레임 조금씩 당겨(lerp, `vx += (targetVx - vx) * turn`) 호밍시킨다.
- **진행 방향 회전 렌더** — 속도 벡터의 각도(`atan2`)로 스프라이트를 돌려 날아가는 방향을 바라보게 한다.
- **명중 VFX + 풀링** — 맞는 순간 판정을 비우고 명중 이펙트를 재생하며, 명중 비트맵은 한 번만 로드해 공유 캐시로 재사용한다.

#### 레이저 빔 (`LaserBeam`)

- **틱 데미지** — 한 번 때리는 탄이 아니라 일정 간격으로 데미지를 여러 번 누적해 주는 지속 빔이다(`tickCooldown -= frameTime`, 0 이하가 되면 데미지 적용 후 간격만큼 리셋).
- **플레이어 추종 원점** — 매 프레임 빔의 시작점을 플레이어 위치로 갱신해 함께 움직인다.
- **적/보스 분기** — 적과 보스를 나눠 처리해 보스에는 같은 빔이 반복해서 들어간다.

**◾ 적 · 진행**

#### 적 4종 (`Enemy`)

한 `Enemy` 클래스가 종류별 스탯을 `enum` 에 묶어 4종을 분기한다. 적 체력은 시간이 갈수록 올라가고, 모두 풀로 재사용되며, 죽을 때 폭발 이펙트와 함께 (분열체를 뺀) 경험치 구슬을 남긴다.

- **자폭 (`SUICIDE`)** — 직진으로 내려오다 일정 높이에 닿으면 플레이어 방향을 단위 벡터로 잡아 가속해(`diveVx = dx / len * diveSpeed`) 그대로 들이받는 돌격병. 약하지만 빠르게 달려든다.
- **원거리 (`RANGED`)** — 정해진 높이(스폰마다 조금씩 다름)에서 멈춰 플레이어를 조준한 탄을 주기적으로 쏘는 견제형.
- **분열 (`SPLIT`)** — 단단한 몸으로 내려오다 죽을 때 구슬 대신 미니언 여러 기를 좌우로 흩뿌린다.
- **미니언 (`SPLIT_MINION`)** — 분열체가 남긴 새끼로, 잠깐 사선으로 흩어졌다가 곧 플레이어 쪽으로 방향을 틀어 추격한다.

#### 시간 기반 스폰 (`EnemyGenerator`)

- **가중치 시간 램프** — 적 종류마다 등장 가중치를 시간에 따라 0에서 최대로 키운다(경과 시간을 등장 구간으로 잘라 비율을 만든다 — `((경과 - 시작) / 구간).coerceIn(0f, 1f)`). 그래서 초반엔 자폭만 나오고, 후반으로 갈수록 세 종류가 골고루 섞인다.
- **위치 랜덤 + 안전 마진** — 스프라이트 폭만큼 여백을 둔 범위에서 가로 위치를 무작위로 잡아 화면 밖으로 잘리지 않게 한다.

#### 적 탄 (`EnemyBullet`)

- **비대칭 히트박스** — 탄 그림이 세로로 길쭉해서 충돌 판정도 가로는 좁게·세로는 덜 좁게 잡아 보이는 모양과 맞춘다.
- **회전 렌더 + 명중 플래시** — 진행 방향으로 회전시키고, 맞으면 짧은 명중 이펙트를 띄운다.
- **풀링 + 공유 비트맵 캐시**

#### 경험치 · 레벨업 흐름 (`ExpOrb` · `LevelUpScene` · `LoadoutCardsView`)

- **자석 구슬** — 적이 떨군 경험치 구슬이 플레이어 방향으로 빨려온다.
- **레벨업 인터럽트** — 경험치가 최대치를 넘는 순간 게임 루프 위에 카드 선택 화면을 올려 진행을 잠시 멈춘다.
- **카드 3장 모달** — 보상 카드 세 장을 띄우며, 글자가 칸을 넘치면 폭에 맞춰 자동으로 줄인다(`drawFitText`). 터치 좌표는 실제 화면 좌표를 게임 좌표로 되돌려(`fromScreen`) 어느 카드를 눌렀는지 가린다.
- **빌드 미니뷰** — 현재 장착한 무기·스킬·능력치 카드와 누적 횟수를 한쪽에 작게 보여준다.

#### 로그라이크 보상 (`RewardCard` · `CardPool`)

- **sealed class 카드** — 능력치·무기·스킬 카드를 한 묶음으로 두고, 고르면 즉시 플레이어에 반영한다.
- **등급 가중 추첨** — 등급마다 뽑힐 가중치(`dropWeight`)를 두고, 전체 가중치 범위에서 난수를 뽑아 거기서 각 가중치를 빼나가다 0 이하가 되는 등급을 고른다(`roll -= dropWeight; roll <= 0 이면 그 등급`). 가중치가 클수록 뽑힐 구간이 넓어 자주 나오고, 그래서 영웅 등급이 가장 드물다.
- **상황 인지 필터** — 이미 가진 스킬이나 더 올릴 수 없는 능력치 카드는 후보에서 빼 무의미한 선택지를 막는다.
- **무기 소비/업그레이드** — 뽑힌 무기 카드는 풀에서 빼고, 상위 등급을 고르면 같은 무기의 하위 등급 카드도 함께 빼 중복을 막는다.

**◾ 스킬**

#### 스킬 시스템 (`Skill` · `SkillButton` · `SkillVFX`)

- **sealed class 3종** — 힐 / 폭발(범위 광역 데미지) / 광폭화(일정 시간 공격·공속 강화) 세 스킬을 각자 다른 발동 효과로 구현했다.
- **4단계 발동 게이트** — 버튼을 눌렀는가 → 스킬을 장착했는가 → 쿨다운이 끝났는가 → 발동 조건(예: 힐은 체력이 안 찼을 때만)을 만족하는가를 차례로 통과해야 발동된다. 버튼을 누른 터치는 플레이어 이동 드래그로 새지 않게 가로챈다.
- **VFX 통합** — 한 번 재생·반복 재생·플레이어 추종(광폭화 오라)을 한 클래스가 모드만 바꿔 처리한다.

**◾ 보스전**

#### 보스 — 상태·페이즈 (`Boss`)

- **Phase 상태머신** — 내려오는 중(`APPROACHING`) → 멈춰서 공격(`ATTACKING`) 두 상태를 오간다.
- **HP 절반 2페이즈 전환** — 체력이 절반으로 떨어지는 순간 한 번만 2페이즈로 바뀌어 그림을 교체하고, 전환 연출 동안은 판정을 비워 무적이 된다.
- **패턴 사이클** — 직전에 쓴 패턴을 빼고 무작위로 다음 패턴을 골라 같은 패턴이 연달아 나오지 않게 하며, 한 패턴은 여러 발을 일정 간격으로 쏘는 버스트로 이뤄진다.

#### 보스 — 탄막 패턴 4종 (`sealed class BossPattern`)

네 패턴을 `sealed class` 로 나눠 같은 틀로 다룬다. 각 패턴은 *쿨다운*(다음 발동까지 간격) · *버스트 수*(한 번 발동에 몇 번 쏘는지) · *버스트 간격* · *fireTick*(한 번 쏠 때 실제로 탄을 만드는 동작)을 페이즈별로 정의하고, 2페이즈에 들어가면 발수·속도·각도를 키워 같은 패턴도 더 매섭게 만든다.

- **CrownShard(산탄)** — 한 번에 여러 발을 부채꼴로 펼친다. 중앙각을 기준으로 등분한 각도마다 `vx = sin(rad)*speed, vy = cos(rad)*speed` 로 방향을 잡아 동시에 발사.
- **EmperorScythe(Bezier 낫)** — Bezier 곡선을 기반으로, 버스트마다 좌우를 번갈아(`fromLeft = tickIndex % 2 == 0`) 시작점을 잡고 제어점을 화면 밖에 둬 플레이어 주변으로 크게 휘어 들어오게 만든 낫탄.
- **AimedBurst(조준 연사)** — 플레이어를 정조준한 탄을 버스트로 연사한다. 발사 방향은 플레이어까지의 벡터를 정규화해 `vx = dx / len * speed` 로 구한다.
- **CorePulse(코어 폭발)** — 코어를 화면 중앙으로 보내고(`vx = (중앙 - 발사점) / 이동시간`), 도착하면 사방 등각(`angle = i * 360 / 개수`)으로 파편을 흩뿌린다.

#### 보스 — 탄 (`BossBullet`)

- **type enum** — 패턴과 페이즈 조합마다 그림·히트박스·명중 이펙트를 데이터로 묶어 둔다.
- **명중 애니메이션** — 맞으면 프레임 시트를 재생하고, 비트맵은 풀과 공유 캐시로 재사용한다.

#### 보스 — 진입·연출 UI

a2dg 의 Scene·Button·Gauge 를 기반 위젯으로 쓰되, 보스전 특유의 화면 구성과 연출은 직접 만들었다.

- **`BossEnterScene`** — 보스 진입 시점마다 게임 위에 반투명 오버레이를 올려 "보스 도전 / 계속 사냥" 을 고르게 한다. 투명 Scene 이라 뒤 게임 화면이 그대로 비치고, '도전' 을 고르면 `BossScene` 으로 넘어간다.
- **`BossWarning`** — 보스 등장 직전 "WARNING" 글자를 일정 간격으로 점멸시키다, 정해진 시간이 지나면 콜백으로 실제 보스를 소환(`spawnBoss`)한다. 등장 직후 잠깐 무적을 줘 즉사를 막는다.
- **`BossScene`** — `MainScene` 을 상속한 보스 전용 화면. 이전 스테이지의 플레이어 상태·점수를 복사해 이어받고, 일반 적 스폰은 끄고 보스만 등장시킨다.
- **보스 HUD** — `BossHpHud`(HP 바, a2dg `Gauge` 기반) / `BossTimerHud`("BOSS STAGE" 표시) / `BossClearLabel`(처치 시 뜨는 클리어 패널).

**◾ 시스템 · 연출**

#### HUD

- **체력·경험치·스탯 라벨** — `PlayerHpHud`·`ExpLabel` 은 a2dg 의 게이지·라벨 위젯으로 좌하단에 그리고, 직접 만든 `DebugStatLabel` 은 현재 공격·공속 배수를 보여주되 버프가 걸리면 빨간색으로 바뀐다.

#### 연출 · 게임 흐름

- **히트스톱 + 화면 흔들림/줌** — 타격감을 위해 직접 넣은 연출. 강한 타격 순간 `update` 를 아주 잠깐 멈춰(히트스톱) 멈칫하게 하고, 동시에 화면을 임의 방향으로 흔들며 살짝 확대했다가, 남은 시간에 비례해 흔들림·줌 세기를 점점 줄여 자연스럽게 가라앉힌다.
- **피격 진동** — 플레이어가 맞으면 안드로이드 `Vibrator` 로 햅틱 진동을 준다.
- **씬 전환** — `MainActivity`(타이틀) → `SkyBlasterActivity`(게임) → `BossScene` → `GameOverScene` 로 이어진다. 그 위에 올리는 일시정지 오버레이(`PauseScene`)는 CookieRun 의 PauseScene 을 차용했다.

## 4. 아쉬운 것들

### 하고 싶었지만 못 한 것들

- **점수 / 클리어타임 랭킹 시스템** — 서버나 로컬 저장 시스템을 구현하여 만들면 재밌었을 거 같다.
- **콘텐츠 다양성** — 보스가 1종(2페이즈)뿐이고 일반 스테이지가 사실상 하나라, 보스·스테이지·적 종류를 더 늘리고 싶었다.
- **빌드 다양성** — 무기 3종 / 스킬 3종 / 능력치 3종 외에도 추가적으로 무기·스킬·시너지 종류를 더 넣고 싶었다.
- **캐릭터 추가** — 캐릭터를 추가한 후 각 캐릭터에게 다른 기본 무기와 스킬을 넣고 싶었으나, 이를 위해서는 무기, 스킬 등이 더 많아야할 거 같아 하지 못했다.
- **불륨 조절** —설정(음량 on/off)을 만들어보고 싶었다.

### (스토어 판매 시) 팔기 위해 보충할 것들

- 음량·진동 설정, 일시정지/재시작 등 편의 기능 보강.
- 저장·이어하기, 업적, 온라인 랭킹 등 편의성 요소.
- 다양한 해상도·기기에서의 QA, 성능 최적화.
- 다양한 수익화 콘텐츠
- 콘텐츠 추가(보스·스테이지·무기 확장)로 플레이 분량 확보.
- 튜토리얼 추가
- 시작 플레이 화면 수정

### 결국 해결하지 못한 문제 / 버그

- *보스 스테이지에 진입할 떄 보스가 내려오는 동안 데미지를 받는 문제 -> 마땅한 해결책을 찾지 못해 임시로 그냥 데미지를 안 입도록 하였지만 더욱 자연스러운 방안이 필요할 거 같다.*
- *유도 미사일의 충돌박스 -> 미사일의 방향이 꺾이면서 유도미사일의 충돌 박스도 꺾여야 하는데 마땅한 방법을 찾지 못했다.*

### 프로젝트를 하면서 겪은 어려움

- **레이저 공격** — 레이지 공격의 애니메이션 재생, 타격 데미지 중첩 및 가시성 부분에서 많은 자료 조사와 시간이 필요했다.
- **스킬 시스템** — 스킬 버튼을 누르면 그쪽으로 이동이 되는 문제나, 폭발 스킬의 데미지가 안 들어가는 등 이번 프로젝트에서 가장 많은 버그가 발생했던 시스템이었다.
- **밸런싱** — 적 스폰 속도·HP 스케일링, 보상 등급 확률, 보스 HP 사이의 난이도 균형을 반복적으로 조정하느라 많은 시간이 필요했다.
